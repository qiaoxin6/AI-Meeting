# 运行时配置改动剧本

## 改动前先确认

- 这是限流参数、Guard 参数、SingleFlight 参数还是线程池参数？
- 改的是全局值还是某个 stage 的局部值？
- 是否有对应的自动生成配置索引需要刷新？

## 改动后至少做三件事

1. 刷新 `xunzhi-ai-runtime/references/generated-config-index.md`。
2. 核对 stage 名称和业务代码里的常量是否一致。
3. 检查改动会不会改变超时、复用边界或并发上限的业务语义。

## 典型风险

- 阈值过小导致误限流。
- 超时过短导致大量长尾失败。
- key 或 TTL 变化导致重复调用复用率突变。
