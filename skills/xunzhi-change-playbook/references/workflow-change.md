# Workflow 改动剧本

## 改动前要知道

- 当前 workflow 被哪个业务场景调用。
- 当前输出字段由哪个 Java 解析器消费。
- 改动是否会影响追问、评分、提题或神态分析链路。

## 改动步骤

1. 先改 YAML 输入输出字段。
2. 再改 Java 解析和 DTO。
3. 重新生成 `generated-workflow-contracts.md`。
4. 检查规则引擎、场景绑定和下游缓存字段是否受影响。

## 易漏项

- 改了 `follow_up_needed` 这类字段名却没改代码。
- 改了开始节点输出字段却没改调用参数映射。
- 改了流程名称却没同步场景绑定或人工文档。
