# 流量限制

`xunzhi-agent.flow-limit` 是最外层流量门禁，和 AI Guard 不是一回事。

## 当前配置

- 全局：`1秒窗口 / 20次`
- 面试答题：`1秒窗口 / 8次`
- 面试重链路：`1秒窗口 / 2次`
- 面试读接口：`1秒窗口 / 15次`
- 面试 AI 调用：`1秒窗口 / 6次`

## 理解方式

- 全局限流防止系统被横向打爆。
- 答题限流保护同一会话被高频提交。
- heavy 限流保护提题、神态分析这类高成本链路。
- read 限流保护恢复页、轮询页不会把缓存打崩。
- ai-call 限流保护模型调用层。

## 易错点

- 限流命中不是 AI 故障，不要误判到 Guard 层。
- 改业务 QPS 时，答题、读取、重链路三类阈值要一起看。
