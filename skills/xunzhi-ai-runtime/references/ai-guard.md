# AI Guard

`xunzhi-agent.ai-guard` 负责做超时、重试、并发上限和熔断保护。

## 全局参数

- 开关：`enable: true`
- 熔断滑窗：`50`
- 失败率阈值：`50%`
- 半开允许调用数：`10`
- 熔断打开等待：`30000ms`
- 执行器线程：`32`

## 分 stage 配置

- `interview-evaluation`：`20000ms / 30并发 / 1次重试 / 100ms等待`
- `interview-followup`：`20000ms / 20并发 / 1次重试 / 100ms等待`
- `interview-extraction`：`60000ms / 8并发 / 不重试`
- `interview-demeanor`：`20000ms / 6并发 / 不重试`

## 业务含义

- 提题比评分更慢，因此超时更长、并发更低。
- 神态分析通常依赖文件输入，因此并发更保守。
- 评分和追问属于交互敏感路径，允许轻量重试。

## 改动原则

- 调大并发前先看线程池、下游模型配额和 singleflight 命中率。
- 调大超时前先排查是否是重复调用、长尾恢复或文件下载导致。
- 新增 stage 时必须同时补配置和调用点常量。
