# 恢复失败排障剧本

## 常见现象

- 恢复页查不到建议、总分或题目。
- 会话显示可恢复，但答题时提示只读。
- 雷达图或总分恢复后比之前更少。

## 先看哪些对象

| 现象 | 先看对象 | 真相源 |
| --- | --- | --- |
| 建议丢失 | `InterviewQuestion`、冷快照 | MongoDB 主记录 / 冷快照 |
| 总分丢失 | 分数聚合、热快照、归档记录 | 运行态聚合 / 快照 / MySQL |
| 可恢复但不能写 | `InterviewSessionRuntimeView`、`loadMode`、`confidence` | 恢复视图 |
| 雷达图变少 | 神态结果、恢复范围、回填逻辑 | 冷快照 / 恢复服务 |

## 优先检查

1. 会话状态是否真的 `canResume`。
2. `InterviewRuntimeRehydrateScope` 用的是哪一种。
3. 缓存、热快照、冷快照、正式记录哪一层缺数据。
4. 查询接口是否有“缓存 -> 数据库回退”的兜底分支。

## 常见根因

- 恢复范围选错，没把需要的材料带回来。
- 快照刷新时机不对，导致热态和正式记录不一致。
- 业务查询只读了缓存，没有走数据库回补。
- 只读恢复被误判成恢复失败。

## 处理建议

- 先确认主记录还在，再确认快照是否完整。
- 如果是展示缺字段，优先查冷快照和主记录；如果是不能继续答题，优先查热快照和 `loadMode`。
- 恢复问题不要只看一个对象，要同时看主记录、快照、归档三层。
