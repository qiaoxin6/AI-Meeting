# 后端架构治理一期实施清单（PRD）

## 1. 文档信息
- 版本：v1.0
- 日期：2026-03-21
- 适用范围：`virtual-character-backend` 后端（`admin` 模块）
- 目标周期：7 个工作日

## 2. 背景与问题
当前系统已可用，但架构层面仍有高优先级风险：
- 多条链路按 `sessionId` 访问资源时，缺少严格 owner 校验，存在越权面。
- WebSocket 语音转写链路存在路径 `userId` 可伪造风险。
- 身份上下文双轨并存（`biz.user.UserContext` 与 `convention.context.UserContext`），历史逻辑含 header 注入用户身份。
- 配置中存在明文敏感信息与可用默认密钥。
- 并发模型不统一（业务类自建执行器、消息序号非原子分配）。

## 3. 一期目标
- 收口全部 P0 安全问题：会话 owner 校验、WebSocket 鉴权、密钥治理。
- 统一身份链路：仅保留 Sa-Token + `@CurrentUser`。
- 稳定核心并发行为：统一线程池入口，修复消息序号并发窗口。
- 建立最小安全回归基线。

## 4. 非目标
- 一期不做全量领域重构。
- 一期不引入新基础设施（例如 MQ 迁移）。
- 一期不重写全量历史接口，仅改高风险链路并保持兼容。

## 5. 实施范围
- Agent 会话与消息链路
- Interview 会话与记录链路
- AI 会话与消息链路
- WebSocket 音频转写链路
- 用户认证与上下文传播链路
- 配置与密钥加载链路

## 8. 验收标准（DoD）
- 安全：
  - 非 owner 访问会话资源必须被拒绝。
  - WebSocket 非法身份连接必须被拒绝。
- 配置：
  - 仓库中不再存在可直接用于生产的明文密钥。
- 稳定性：
  - 并发下无消息序号冲突。
  - 业务链路不再自建临时执行器。
- 质量：
  - 新增安全回归测试全部通过。
  - CI 编译与测试全部通过。

## 9. 风险与回滚
- 风险 1：前端依赖旧行为，改造后出现兼容问题。
  - 对策：先灰度“告警模式”，再切换为强拦截。
- 风险 2：历史脏数据不满足新 owner 规则。
  - 对策：准备数据修复脚本与迁移窗口。
- 风险 3：WebSocket 鉴权增强影响本地联调效率。
  - 对策：仅在本地 profile 提供调试开关（默认关闭）。
